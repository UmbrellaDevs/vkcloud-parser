# Парсер серверов VK Cloud

Автоматический поиск серверов с нужной IP-подсетью в VK Cloud.
Веб-лаунчер с мультиаккаунтом, профилями поиска и Telegram-уведомлениями.

## Требования

- **Windows 10/11** (или Linux)
- **Python 3.10+**
- **Node.js 18+**
- **OpenSSH** (обычно предустановлен)

## Установка требований

### Python

1. Скачайте с https://www.python.org/downloads/
2. Запустите установщик
3. **ВАЖНО:** на первом экране поставьте галочку **"Add Python to PATH"** (внизу)
4. Нажмите **"Install Now"**
5. Дождитесь установки, нажмите Close
6. Проверьте: откройте cmd и введите `python --version` — должна показать версию

### Node.js

1. Скачайте LTS версию с https://nodejs.org/ (кнопка слева — LTS)
2. Запустите установщик
3. Жмите **Next** на всех шагах, ничего менять не нужно
4. На шаге "Tools for Native Modules" — поставьте галочку **"Automatically install..."**
5. Нажмите **Install**, дождитесь завершения
6. Проверьте: откройте cmd и введите `node --version` — должна показать версию

### OpenSSH (обычно уже есть)

Проверьте: откройте cmd и введите `ssh -V`. Если показывает версию — всё ок.
Если нет — откройте **Параметры → Приложения → Дополнительные компоненты → Добавить компонент** → найдите **OpenSSH Client** → Установить.

## Быстрый старт

### 1. Установка Python-зависимостей

Откройте cmd в папке с парсером и введите:

```
pip install -r requirements.txt
```

### 2. Установка Node.js-зависимостей

```
cd launcher
npm install
cd ..
```

### 3. Настройка ProxySoxy (авто-закупка прокси)

Для работы авто-закупки прокси нужен API-ключ сервиса ProxySoxy:

1. Зарегистрируйтесь на https://proxysoxy.com/
2. Пополните баланс
3. Перейдите в раздел API — скопируйте ваш API-ключ
4. Откройте файл `launcher/.env` и заполните:
   ```
   PROXYSOXY_API_KEY=ваш_ключ_сюда
   ```

Без ключа авто-закупка прокси работать не будет, но вы сможете указывать прокси вручную.

### 4. Первый запуск

Запустите `start.bat` — откроется веб-панель на `http://localhost:3847`

## Добавление аккаунтов VK Cloud

### Подготовка

1. Зарегистрируйтесь на https://cloud.vk.com/
2. Привяжите платёжную карту к проекту
3. **Включите 2FA** (обязательно для доступа к API)
4. Перейдите в **Настройки проекта → Доступ по API** → включите доступ
5. Скопируйте **Project ID** из настроек проекта (или из файла openrc v3, переменная `OS_PROJECT_ID`)

### Добавление в панель

1. В веб-панели нажмите **Добавить аккаунт**
2. Заполните:
   - **Название** — произвольное имя (например, email)
   - **Email** — email аккаунта VK Cloud
   - **Пароль** — пароль от аккаунта
   - **Project ID** — из файла openrc (переменная `OS_PROJECT_ID`)
   - **SOCKS5 прокси** — формат `user:pass@host:port` (рекомендуется)
3. Нажмите **Сохранить**

### 5. Настройка поиска

В веб-панели в разделе **Конфигурация**:

- **Целевые подсети (CIDR)** — укажите нужные подсети через запятую. Поддерживаются форматы:
  - Префикс: `95.163`
  - CIDR: `95.163.248.0/22`
  - Несколько: `95.163, 185.6.176.0/22`
- **Профиль поиска** — выберите режим работы парсера

### 6. Запуск парсера

Нажмите **Запустить** — парсер начнёт искать IP с нужной подсетью.

## Профили поиска

| Профиль | Описание |
|---------|----------|
| **Быстрый** | Минимальные задержки, 10 попыток за сессию, пауза 1-2 мин. Быстрый поиск, выше риск бана |
| **Сбалансированный** | Средние задержки, 20 попыток за сессию, пауза 1-4 мин. Оптимальный баланс |
| **Безопасный** | Большие задержки, 15 попыток за сессию, длинные паузы. Минимальный риск бана |
| **Без пауз** | Без задержек и пауз между сессиями. Максимальная скорость, максимальный риск |
| **Тестовый** | 3 попытки, быстрые паузы. Для проверки работоспособности |

## Как это работает

1. Авторизуется через Keystone (OpenStack) — получает токен
2. Создаёт floating IP в VK Cloud
3. Проверяет IP-адрес — совпадает ли с целевыми подсетями (CIDR)
4. Если не совпал — удаляет IP (в фоне, с задержкой) и создаёт новый
5. Если нашёл — создаёт VM, настраивает SSH, генерирует пароль, отправляет в Telegram
6. Работает сессиями: N попыток → пауза → следующая сессия

## Функции лаунчера

- **Мультиаккаунт** — запускайте парсер на нескольких аккаунтах одновременно
- **Профили поиска** — 5 готовых режимов с разным балансом скорости и безопасности
- **CIDR-матчинг** — точный поиск по подсетям (поддержка CIDR и префиксов)
- **Фоновое удаление IP** — неподходящие IP удаляются в отдельных потоках
- **Сессии и паузы** — автоматические перерывы для снижения риска блокировки
- **Авто-обновление токенов** — Keystone-токены обновляются автоматически
- **Авто-закупка прокси** — SOCKS5 прокси через ProxySoxy одной кнопкой
- **Telegram-бот** — уведомления о найденных серверах
- **Веб-терминал** — подключение к найденным VM прямо из браузера
- **Логи в реальном времени** — SSE-стриминг логов каждого аккаунта
- **SQLite БД** — надёжное хранение данных

## Telegram-бот (встроенный)

Бот запускается автоматически вместе с лаунчером.

Для работы бота укажите `telegram_bot_token` и `telegram_chat_id` в настройках лаунчера.

## Настройка Telegram-бота

1. Создайте бота через [@BotFather](https://t.me/BotFather) — скопируйте токен
2. Узнайте свой chat_id — напишите `/start` боту [@userinfobot](https://t.me/userinfobot)
3. В веб-панели в разделе **Конфигурация** заполните:
   - **Telegram Bot Token** — токен бота
   - **Telegram Chat ID** — ваш chat_id
4. Нажмите **Сохранить** и перезапустите лаунчер

## Подсети VK Cloud

| Подсеть | Описание |
|---------|----------|
| `95.163` | Основная подсеть VK Cloud (LLC VK, AS47764) |

По умолчанию парсер ищет `95.163`.
Можно указать несколько подсетей через запятую в поле "Целевые подсети (CIDR)" в веб-панели.

## Зоны доступности VK Cloud

| Зона | Расположение |
|------|-------------|
| `MS1` | Москва (DataLine NORD4) |
| `GZ1` | Москва (Гознак) |
| `ME1` | Москва (Ростелеком) |
| `QAZ` | Казахстан |

## Прокси

Рекомендуется использовать SOCKS5 прокси для каждого аккаунта — это снижает риск блокировки.

Формат: `user:pass@host:port`
